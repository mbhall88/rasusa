# Phase 5.2 Research: SBC Refinements and Fixes

## Goal
Research how to implement Phase 5.2 (SBC Refinements and Fixes) based on `05.2-CONTEXT.md` to prepare an actionable plan.

## Technical Context

### 1. `output_type` to `compress_type` CLI Renaming
In `src/reads.rs`, `Reads` struct currently has:
```rust
    #[clap(short = 'O', long, value_name = "u|b|g|l|x|z", value_parser = parse_compression_format)]
    pub output_type: Option<niffler::compression::Format>,
```
We need to:
- Rename `output_type` to `compress_type`.
- Change short flag from `-O` to `-Z` (or similar).
- Change long flag to `--compress-type`.
- Update the instances of `self.output_type` to `self.compress_type` in `src/reads.rs`.
- We should also check `src/alignment.rs` to see if `output_type` flag is there. It is present:
```rust
    #[arg(short='O', long, value_name = "FMT", value_parser = infer_format_from_char)]
    pub output_type: Option<Format>,
```
In `Alignment` it really represents the SAM/BAM/CRAM format. We might want to rename it to `--output-format` to align with the new flag in `Reads`.

### 2. New `--output-format` Flag
For `rasusa reads`, we need a new explicit `--output-format` option to override default inferred output behavior.
- Expected values: `fasta`, `fastq`, `sam`, `bam`, `cram`.
- We need to create an enum `OutputFormat` in `src/cli.rs` that can be parsed by `clap`.
- Then add `output_format: Option<OutputFormat>` to `Reads` struct with short `-O` and long `--output-format`.

### 3. Fixing FASTA Output in `AlignmentSource`
Currently, `filter_reads_into` in `AlignmentSource` (`src/source.rs`) has the signature:
```rust
    fn filter_reads_into(
        &self,
        reads_to_keep: &[bool],
        nb_reads_keep: usize,
        write_to: &mut dyn Write,
        output_format: Option<Format>,
    ) -> Result<usize, FastxError>;
```
The `Format` type here comes from `noodles_util::alignment::Format`. It cannot represent FASTA/FASTQ.
To support forcing FASTA, we can update `output_format` parameter to be an internal `OutputFormat` enum that covers both Sequence (Fasta, Fastq) and Alignment formats, or we can pass a boolean `force_fasta` to `filter_reads_into` (and potentially other `RecordSource` implementors).
If `qual.is_empty()`, it outputs FASTA. But if `force_fasta` is true, it should output FASTA even if `qual` is present.

### 4. Documentation
We need to update:
- `README.md`
- `src/reads.rs` CLI help strings
- Any docs referring to "SAM/BAM/CRAM" in `reads` command to qualify it as "*unaligned* SAM/BAM/CRAM".

- Integration tests confirm the `--compress-type` flag works.

## Summary for Planner
1. Update `src/cli.rs` to include a new `OutputFormat` enum (`fasta`, `fastq`, `sam`, `bam`, `cram`).
2. Update `src/reads.rs`: rename `--output-type` to `--compress-type` (flag `-Z`), and add `--output-format` (flag `-O`).
3. Update `src/alignment.rs`: rename `--output-type` to `--output-format` for consistency.
4. Update `src/source.rs` (and `RecordSource` trait in `fastx.rs` if needed) to take `OutputFormat` or a `force_fasta` flag, adjusting the FASTA writing block to skip FASTQ output if FASTA is requested.
5. Update docs to clarify "unaligned" SAM/BAM/CRAM.
